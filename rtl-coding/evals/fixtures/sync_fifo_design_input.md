# sync_fifo 模块设计输入

本文件是已冻结、可直接编码的模块设计输入，供 RTL 编码使用。所有接口、时序、异常与验证关注点均已确定，无需再做架构或微架构决策。

## 1. 模块职责

- `sync_fifo`：单时钟同步 FIFO，缓存写入侧数据，按先进先出顺序供读出侧取用。
- 非职责：不做跨时钟域同步，不做数据变换，不含 CBB/IP 复用（存储自实现）。

## 2. 参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `DATA_WIDTH` | 8 | 数据位宽 |
| `DEPTH` | 16 | 深度，取 2 的幂 |
| `ADDR_WIDTH` | 4 | 地址位宽，等于 log2(DEPTH) |

## 3. 时钟与复位

- 单时钟 `clk`，上升沿有效。
- 异步复位 `rst_n`，低有效；复位期间指针、计数清零，输出回到 `empty=1`、`full=0`、`rd_data=0`、`count=0`。

## 4. 接口

| 端口 | 方向 | 位宽 | 说明 |
| --- | --- | --- | --- |
| `clk` | input | 1 | 时钟 |
| `rst_n` | input | 1 | 异步低有效复位 |
| `wr_en` | input | 1 | 写请求，高有效 |
| `wr_data` | input | DATA_WIDTH | 写数据 |
| `rd_en` | input | 1 | 读请求，高有效 |
| `rd_data` | output | DATA_WIDTH | 读数据，寄存器输出 |
| `full` | output | 1 | 满标志，`count == DEPTH` 时为高 |
| `empty` | output | 1 | 空标志，`count == 0` 时为高 |
| `count` | output | ADDR_WIDTH+1 | 当前存储条目数 |

## 5. 主功能行为

- 标准同步 FIFO（非 first-word-fall-through）。读出为寄存器输出：当 `rd_en && !empty`，`rd_data` 在下一拍更新为当前读指针指向的数据。
- 写接受条件：`wr_en && !full`，数据写入写指针位置，写指针 +1。
- 读接受条件：`rd_en && !empty`，读指针 +1。
- 同拍读写且既不空也不满时，两侧各自推进，`count` 不变。
- `full` 在 `count == DEPTH` 时有效；`empty` 在 `count == 0` 时有效。
- 存储：内部寄存器数组 `mem[0:DEPTH-1]`，自实现，不使用 CBB/IP。

## 6. 边界与异常

- 满时写（`wr_en && full`）：忽略该写请求，不覆盖已存数据，写指针与 `count` 不变。
- 空时读（`rd_en && empty`）：忽略该读请求，`rd_data` 保持上一次值，读指针与 `count` 不变。
- 无错误上报端口；异常仅表现为忽略非法请求。

## 7. 实现约束

- 吞吐：读写各 1 次/拍。
- 输出寄存器化：`rd_data`、`full`、`empty`、`count` 均为寄存器时序输出。
- 存储自实现，禁止替换为未确认的外部 IP。

## 8. 验证关注点

- 写满到 `full`、读空到 `empty` 的边界跳变。
- 同拍读写时 `count` 保持不变。
- 满时写被忽略、空时读被忽略。
- 复位在操作中途拉低后的状态恢复。
