# 图表风格迁移

在用户提供参考图并要求「按这张图的风格」时使用。参考图只提供视觉语法；新图内容必须来自已有设计输入，并按芯片设计文档风格渲染/排版；非 RTL 通用图才切回 `generation.md`。

## 使用时机

- 用户上传或贴出图片，并要求「按这个风格画」「参考这张图的样式」「照着这个排版/配色画」。
- 新图内容来自用户提供的模块/接口/路径/时钟域说明、项目代码、RTL、寄存器/接口说明或已有草图。
- 参考图可以来自论文、开源 RTL 文档、规范说明或用户草图；不要写死为某一种来源或某一张图。

若只有参考图而没有新图设计内容，先追问要渲染的已有设计；不要从参考图推导新模块或新连接。

## 工作流程

### Step 1：提取视觉规则

只提取视觉规则，不迁移参考图原始业务内容：

- 画布比例、留白、标题位置、分区标题风格
- domain/stage/subsystem 边界方式和低饱和配色
- 模块形状、边框、字体、字号、文字密度
- 连线粗细、箭头、正交路径、反馈线走法
- 信号/总线标签的位置和文字风格

### Step 2：整理已有设计输入

从新图输入中整理已给定的模块、端口、边界、连接、数据/控制路径、反馈路径和时钟域。不要复制参考图中的节点名、实体名、CPU-specific 模块名或协议细节，除非它们确实属于新图设计。

### Step 3：风格与内容对齐

- 用参考图的视觉语言排版新图，但模块和连接必须来自新图输入。
- 内容结构仍遵守 `tech-diagram.md`：左右数据流、控制/数据路径分离、必要的 domain/boundary、外侧反馈路径和独立标签。
- 若参考图是彩色架构海报或 PPT 风格，只保留其可读性和排版优点，避免退化成装饰图。
- 连线文字仍按主规则处理：edge `value` 为空，文字使用独立且透明背景的 text vertex，不使用参考图里的 edge label 或白底遮线手法。

### Step 4：生成与验证

生成完整 `.drawio` XML，保存后运行：

```bash
python <drawio-chip skill directory>/scripts/validate_drawio.py --no-edge-labels --transparent-text-labels --no-text-line-overlap <file.drawio>
```

验证失败先修复再交付。

## 注意事项

- 只迁移风格，不迁移参考图内容。
- 内容必须能追溯到已有设计输入或明确假设。
- 提炼参考图中的可复用视觉特征，例如布局方向、层级分组、线型、配色语义、标签方式和留白比例；新图内容仍以当前输入设计为准。
- 最终输出说明风格摘要、内容来源、文件路径、验证结果、使用说明和假设。
