# 技术图表参考

用于生成芯片设计类 Draw.io 图表：芯片架构、模块关系、控制流程、数据流、状态机、片上接口交互、总线事务等。先识别图表意图，再选择合适的芯片设计结构。

## 使用时机

- 用户要求绘制芯片架构、模块关系、数据流、控制流程图、状态机图、片上接口交互或总线事务图
- 用户提供一组模块、接口、信号、寄存器、状态或依赖关系，需要整理成 `.drawio`
- 用户没有提供参考图，只描述了芯片设计图的内容与关系

## 工作流程

### Step 1：明确图的核心信息

1. **确定图表目标**：解释芯片结构、展示控制流程、说明数据流、表达模块关系、描述片上接口或总线事务等。
2. **列出关键元素**：模块、分组、外设接口、寄存器/存储、输入输出信号、关键约束。
3. **梳理关系类型**：数据流、控制流、时钟/复位、握手、依赖、包含、顺序、条件分支。
4. **决定抽象层级**：一张图只表达一个层级；不要把顶层架构、控制流程、接口时序混在同一张图里。

### Step 2：先设计布局，再写 XML

1. **选择布局方向**：
   - 左到右：适合数据流、总线通路、处理链路
   - 上到下：适合控制流程、状态决策、仲裁流程
   - 分层布局：适合芯片架构、模块层级、时钟/复位域
   - 中心发散：适合 NoC、crossbar、多接口互连
2. **规划画布与间距**：常用画布 800–1200 宽、500–900 高；节点间距至少 30px；容器内边距至少 30px。
3. **规划连线路径**：主路径尽量直，辅助关系用虚线；外侧跳连避免穿过节点文字。
4. **确认阅读顺序**：从入口到出口、从上层到下层、从左到右保持一致。

### Step 3：使用统一样式

- **节点**：圆角矩形为主，特殊语义才使用菱形、圆柱、云、文件等形状。
- **颜色**：同类节点同色；颜色控制在 4–6 种以内。
- **字号**：节点标题 12–14px，说明文字 10–12px。
- **连线**：主路径 `strokeWidth=2`，辅助路径虚线或淡色。
- **分组**：用浅色容器或 swimlane 表示边界，避免过度装饰。

### Step 4：生成 XML（关键）

1. 先写 `<mxfile>`、`<diagram>`、`<mxGraphModel>`、`<root>` 基础结构。
2. 固定根节点：`<mxCell id="0"/>` 与 `<mxCell id="1" parent="0"/>`。
3. 先生成容器，再生成节点，最后生成连线与注释，保证层级顺序正确。
4. 每个节点必须有 `vertex="1" parent="1"` 和完整 `mxGeometry`。
5. 每条连线必须有 `edge="1" parent="1"`，并通过 `source`/`target` 指向已有节点。
6. 不要压缩 XML；保持缩进，便于用户继续修改。

### Step 5：添加辅助表达

- 输入/输出信号标签、编号、条件说明、数据宽度、异常/超时分支说明
- 图例、阶段标题、分组标题、外设接口标记
- 虚线辅助连接、浅色背景区、注释框、芯片/子系统边界说明

## 通用图表类型

### 芯片架构图

**适用**：展示芯片顶层模块、子模块、总线/互连、接口、存储与时钟/复位域之间的关系。

**结构**：
- 顶层芯片边界
- 处理/控制模块
- 总线、NoC 或互连
- 外设接口
- 寄存器、SRAM、FIFO 等存储单元
- 时钟域、复位域或电源域

**画法**：
- 用大容器表示芯片或子系统边界
- 数据路径尽量左到右，控制路径可上到下
- 总线/互连用较粗箭头或带标签连线表示
- 时钟/复位域用浅色分组、虚线边界或标签区分

**样式约定**：
- 芯片/子系统边界：浅灰或浅黄容器，标题放左上角
- 计算/控制模块：浅蓝圆角矩形，同层级尺寸尽量一致
- 寄存器/SRAM/FIFO：浅绿圆柱、存储形状或带 `REG`/`MEM` 标签的矩形
- 外设/IO 接口：浅红卡片，贴近芯片边界或外侧连接点
- 总线/NoC/互连：粗实线、中心横条或带标签箭头，标注位宽、方向和协议名
- 时钟/复位/电源域：虚线分组或淡色背景，避免和功能模块使用同色

### 控制流程图

**适用**：展示数字设计中的控制步骤、仲裁流程、条件分支、异常/超时路径。

**结构**：
- 开始/结束：圆角矩形
- 控制步骤：矩形或圆角矩形
- 判断/仲裁：菱形
- 输入/输出条件：平行四边形或普通矩形
- 异常/超时分支：虚线或灰色箭头

**画法**：
- 主控制路径保持一个方向
- 分支箭头标注条件
- 循环回边避免穿过主要节点
- 节点文字使用短动词或控制动作名

### 数据流图

**适用**：展示数据从输入接口、处理模块、寄存器/存储到输出接口的路径。

**结构**：
- 输入接口或上游模块
- 处理/转换模块
- 寄存器、SRAM、FIFO 或 Buffer
- 输出接口或下游模块
- 信号名、位宽、格式/字段、valid/ready 条件

**画法**：
- 箭头标签写信号名、位宽、数据格式或有效条件
- 寄存器/存储可用圆柱形、存储形状或带 `REG`/`MEM` 标签的矩形
- 处理模块用统一颜色，存储节点用另一种颜色
- 主数据流保持单向，避免多方向交叉

### 数字设计状态机图

**适用**：展示 RTL/数字设计中的状态集合、复位入口、转移条件与状态输出动作。

**结构**：
- 初始/复位状态
- 工作状态与等待状态
- 状态转移条件
- 输入触发与输出动作
- 错误、超时或结束状态

**画法**：
- 状态节点按主循环顺序排列，避免箭头交叉
- 转移箭头标注布尔条件或事件名
- 复位路径用单独入口箭头标清楚
- 状态输出可写在节点内，复杂动作放在旁注框

**样式约定**：
- 普通状态：浅蓝圆角矩形或圆形节点
- 初始/复位状态：浅绿节点、双边框或入口箭头强调
- 错误/异常状态：浅红节点，避免和普通状态同色
- 完成/终止状态：浅紫节点或粗边框节点
- 普通转移：实线箭头，`endArrow=classic`
- 复位/异步跳转：虚线箭头，标签写 `reset`、`timeout` 或异常原因
- 箭头标签：统一使用 `condition / action` 或 `event -> output`，不要在同一图混用多种格式

### 片上接口 / 总线事务图

**适用**：展示模块之间的片上接口关系、总线事务、握手流程或请求/响应通路。

**结构**：
- 主设备 / 发起模块
- 从设备 / 响应模块
- 请求、响应、数据通道
- valid/ready、req/ack、addr/data/control 等信号
- 错误、重试、等待或 backpressure 路径

**画法**：
- 请求通道用实线箭头，响应或返回数据可用反向箭头
- valid/ready、req/ack 等握手信号写在箭头标签上
- 多通道接口可用并行箭头或分组容器表示
- 可用编号标注事务顺序

## 通用样式

### 基础节点

```xml
<mxCell id="node" value="模块" style="rounded=1;whiteSpace=wrap;html=1;fillColor=#DAE8FC;strokeColor=#6C8EBF;fontSize=13;" vertex="1" parent="1">
  <mxGeometry x="120" y="80" width="120" height="50" as="geometry"/>
</mxCell>
```

### 分组容器

```xml
<mxCell id="group" value="子系统" style="swimlane;html=1;startSize=28;rounded=1;fillColor=#F5F5F5;strokeColor=#999999;fontStyle=1;" vertex="1" parent="1">
  <mxGeometry x="80" y="60" width="320" height="220" as="geometry"/>
</mxCell>
```

### 正交箭头

```xml
<mxCell id="edge" value="调用" style="edgeStyle=orthogonalEdgeStyle;rounded=0;orthogonalLoop=1;jettySize=auto;html=1;strokeWidth=2;endArrow=classic;" edge="1" parent="1" source="a" target="b">
  <mxGeometry relative="1" as="geometry"/>
</mxCell>
```

### 虚线辅助关系

```xml
<mxCell id="dashed" value="可选" style="edgeStyle=orthogonalEdgeStyle;rounded=0;html=1;dashed=1;strokeColor=#666666;endArrow=classic;" edge="1" parent="1" source="a" target="b">
  <mxGeometry relative="1" as="geometry"/>
</mxCell>
```

## 设计原则

- **先结构后样式**：先确定节点、分组、箭头关系，再决定颜色和形状。
- **同类一致**：同类节点保持相同尺寸、颜色、字体和边框。
- **主路径清晰**：主流程/主数据流使用实线粗箭头；辅助关系用虚线或淡色。
- **减少交叉**：优先用分层布局和正交连线，必要时增加中继节点。
- **文字简短**：节点标题短，细节放在注释框或箭头标签中。
- **边界明确**：芯片边界、子系统边界、外设接口、寄存器/存储、异步时钟域要视觉区分。

## XML 生成检查清单

- [ ] 每个 `<mxCell>` 都有对应的 `</mxCell>`，不能写成 `</mCell>`
- [ ] 节点使用 `vertex="1"`，连线使用 `edge="1"`
- [ ] 所有 ID 唯一，连线的 `source` 和 `target` 指向存在节点
- [ ] `mxGeometry` 的 `x/y/width/height` 合理，不出现负尺寸
- [ ] 所有普通元素都有 `parent="1"`，根元素除外
- [ ] 节点不重叠，连线不穿过关键文字，外侧跳连留足空间
- [ ] 同类节点的宽高、字号、圆角、描边保持一致
- [ ] `value` 属性里的富文本标签必须按 XML attribute 规则转义，例如写成 `&lt;font style=&quot;...&quot;&gt;文本&lt;/font&gt;`，不要直接写原始 `<font>` 标签
- [ ] 分组容器的底边/顶边必须留出至少 30px 内边距，确保最靠近边界的节点完全在容器内而不越界

## 图表质量检查清单

- [ ] 图中每个节点都有明确角色，能追溯到用户描述
- [ ] 箭头方向符合实际关系，主路径一眼可辨
- [ ] 同一层级对齐，间距一致，分组边界清晰
- [ ] 分组容器没有遮挡内部节点，标题不压住内容
- [ ] 颜色含义一致，且不依赖颜色传达唯一信息
- [ ] 节点文字短而清晰，细节放在注释框或箭头标签中
- [ ] 输出未压缩、可读 XML，后续可在 Draw.io 中继续编辑

## 输出说明

输出时说明：图表用途、核心组件、布局与配色、文件位置、Draw.io 打开方式、需要用户确认的假设。

## 常见问题

- "Not a diagram file"：检查 `<mxfile>`、`<diagram>`、`<mxGraphModel>` 的嵌套关系
- "Opening and ending tag mismatch"：确保所有 XML 标签闭合正确
- 节点/连线不显示：检查 `vertex`/`edge`、`parent`、`source`/`target`
- 中文乱码：使用 UTF-8 编码，特殊字符按 XML 规则转义
- 图看起来不专业：优先检查字号是否过小、模块是否拥挤、颜色是否过多、连线是否穿过主体
