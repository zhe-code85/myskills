# Visual Checklist

当你需要做版式视觉验收、视觉复核、排版漂移分析、`visual-check.md` 或 `layout-fix-list.md` 时，读取并执行本文件。

## 验收前提

- 已完成输入门禁、模型校验、Stage-1 模板保真和 Stage-2 模板保真。
- 模板 DOCX 和生成稿 DOCX 都要渲染；不得只看生成稿。
- 模板保真存在 Blocker 时，视觉验收直接记为 failed，不得写 passed。
- 只看 contact sheet、只抽样、或只检查截图，不能写 passed；最多写 `accepted-with-notes`、`blocked` 或 `not run`。
- 运行或等价执行 `render_and_check.py`，至少扫描可见 TOC / PAGEREF 域代码、旧模板词、空白页、断裂或被拉散的 `DS_*` 标记。脚本通过不代表视觉验收通过；脚本失败则本轮视觉验收不得写 passed。

## 验收循环

1. 渲染模板关键页和生成稿关键页。
2. 按页面角色执行检查。
3. 输出或更新 `visual-check.md`。
4. 若结论为 `passed`，进入最终交付。
5. 若结论为 `failed`，生成 `layout-fix-list.md`。
6. 按严重度修复版式问题。
7. 重新生成 DOCX。
8. 重新渲染并复查。
9. 重复循环，直到收敛到允许的停止条件。

允许的停止条件：

| 状态 | 含义 |
| --- | --- |
| `passed` | 已按要求完成视觉验收，无阻断问题 |
| `accepted-with-notes` | 仍有差异，但用户明确接受，并记录剩余风险 |
| `blocked` | 缺少渲染工具、字体、授权资产、模板损坏或无法稳定复现 |
| `not run` | 无法执行视觉验收；不能声称版式通过 |

## 检查页范围

页数不超过 20 页时必须逐页检查。超过 20 页时至少检查：

- 首页 / 封面。
- 目录页。
- 普通正文页。
- 双栏正文页，如模板存在。
- 参数表页。
- 图文混排页。
- Package / Mechanical / Tape & Reel 页，如存在。
- Important Notice / 法务 / 支持页。
- 每个含图页。
- 每个长表页。
- 所有上一轮出错页。

`visual-check.md` 必须列出渲染页数、实际检查页码、未检查页码及原因。

## 视觉 Checklist

- [ ] 页面尺寸、方向、页边距与模板一致。
- [ ] 页眉页脚位置、内容、横线、页码样式和 section 继承关系与模板一致。
- [ ] 公司名、Logo、水印、版权、文档状态标识位置正确。
- [ ] 封面 / 首页固定信息没有丢失，产品型号、标题、状态、版本、日期可读。
- [ ] 标题层级、编号、字体、段前段后间距符合模板。
- [ ] 目录保留 Word TOC 字段或用户接受的替代机制；目录样式、缩进、点线、页码位置符合模板。
- [ ] 表格宽度未超出页面，列宽、边框、表头样式符合模板。
- [ ] 强结构化表格没有退化成用空格或 tab 对齐的伪表格。
- [ ] 长表格跨页仍可读，必要时重复表头。
- [ ] 图片未变形、未越界、未遮挡文字。
- [ ] 图片、图表或占位图保留模板容器、caption 和上下文说明，没有整页只有占位文字。
- [ ] Figure caption / Table caption 风格和位置符合模板。
- [ ] Note / Notes 紧贴对应表格、图片或参数块，未遗漏。
- [ ] 没有孤立标题、异常空白页、残留空表格行、半页以上异常空洞或明显分页错误。
- [ ] Important Notice、法务声明、支持页视觉风格保留。
- [ ] `DS_*` 标注在草稿版醒目但不破坏正文、表格、caption 或页眉页脚。
- [ ] clean 版不得残留未允许的 `DS_*` 标注。
- [ ] Draft Review Notes、差异清单或待确认清单位于用户接受的位置，不能像调试附录一样破坏模板后置页。
- [ ] release 版无未授权竞品图片或竞品截图残留。
- [ ] 旧模板示例内容没有残留，特别是与目标封装冲突的 pinout、package、DFN / SOT 示例图、旧 pin 表、旧应用图、旧公司主体或旧产品型号。
- [ ] `PIN CONFIGURATION` 与 `PIN DESCRIPTION` 视觉一致；目标封装 top-view、pin 编号、pin name、DAP / Thermal Pad 和状态标注没有被旧模板图表误导。

## 内容长度与排版漂移

基于模板 DOCX 复制生成时，内容长度变化允许导致页码和页数变化，但不得破坏模板版式规则。必须检查：

- [ ] 首页核心信息没有被挤出首屏或覆盖 Logo、状态标识、法务区域。
- [ ] Description / Features / Applications 等首页或前置区块没有互相挤压、重叠或截断。
- [ ] 长段落已拆成短段落、bullet 或 Note，而不是靠缩小字体硬塞。
- [ ] 长 bullet list 没有导致栏间溢出、跨栏错位或异常换页。
- [ ] 双栏正文中左右栏流动合理，没有大面积空洞或内容跳栏异常。
- [ ] 标题与其后第一段、表格、图片保持在同页或合理位置，避免孤立标题。
- [ ] 表格因行数增加跨页时仍可读，表头重复，Notes 不与表格主体分离。
- [ ] 参数表列宽没有因长参数名、条件文本、单位或 Note 撑破页面。
- [ ] 图片和 caption 没有因为前文变长而被挤压、错位或脱离上下文。
- [ ] Note / Notes 与对应表格、图片、参数块保持邻近关系。
- [ ] Important Notice、Package、Mechanical、Tape & Reel 等后置章节没有被前文分页破坏样式。
- [ ] 页眉页脚没有因为 section break、分页变化或内容增减而丢失、错继承或错页。
- [ ] 目录页码可变化，但目录样式、缩进、点线、页码对齐必须保持。
- [ ] 图号、表号、交叉引用和目录在内容变化后已更新或标注需更新。
- [ ] 目录仍是真实可更新 Word TOC 字段，或用户明确接受普通文本目录；隐藏 TOC marker 不算保真。
- [ ] PDF / 文本扫描无可见 `TOC \h`、`PAGEREF`、`MERGEFORMAT` 等域代码。
- [ ] `DS_*` 标记没有被拉散成 `D S _...`、截断成 `DS_...` 或在表格中断词。
- [ ] 没有异常空白页、半页大空洞、页面底部孤立一行、表格后 Note 跑到下一页顶部等明显排版问题。

发现内容长度导致排版问题时，优先按以下顺序收敛：调整内容结构，调整表格结构，调整插入位置，调整分页控制，最后才考虑轻微样式调整。不得通过随意缩小字体、压缩行距或破坏模板样式来硬塞内容；如果压缩内容会改变技术含义，必须向用户确认。

## 问题分级

| 级别 | 含义 | 处理 |
| --- | --- | --- |
| Blocker | 影响交付或误导读者 | 必须修复或用户明确接受 |
| Major | 明显破坏模板风格、可读性或专业性 | 默认必须修复 |
| Minor | 轻微差异，不影响阅读和专业性 | 可记录为可接受差异 |
| Info | 观察项或后续优化 | 不阻塞交付 |

## visual-check.md 记录格式

每轮追加记录：

- 轮次编号。
- 输入 DOCX。
- 模板渲染产物路径。
- 生成稿渲染产物路径。
- 渲染工具和字体环境。
- 总页数、检查页码、未检查页码及原因。
- 页面角色覆盖情况。
- 发现问题列表，含严重度、页码、证据路径、原因判断。
- 修复动作或建议。
- 复查结果。
- 本轮结论：`passed`、`failed`、`accepted-with-notes`、`blocked` 或 `not run`。

`layout-fix-list.md` 至少列出问题编号、严重度、页码、现象、疑似原因、修复策略、负责人 / 执行者、复查状态。
